```bash

# 2. Git 初期化
```

## 5. API実装（analyze / parse-url）

### 実装ファイル

| ファイル | 内容 |
|---|---|
| `app/lib/websiteParser.ts` | HTML取得・本文抽出・タイムアウト8s・プライベートIP遮断 |
| `app/app/api/parse-url/route.ts` | POST /api/parse-url → URL→LLM要約→事業内容テキスト返却 |
| `app/app/api/analyze/route.ts` | POST /api/analyze → スコア計算→LLM文章生成→DiagnosisResult返却 |

## 6. Netlify デプロイとトラブルシューティング

### netlify.toml の構成決定

| ファイル | 対象サイト | 内容 |
|---|---|---|
| `netlify.toml`（リポジトリルート） | ai-tool-diagnosis-2（**作業対象**） | `base="app"` を指定 |
| `app/netlify.toml` | ai-tool-diagnosis（旧サイト・放置） | `base="app"` + `[[plugins]] @netlify/plugin-nextjs` |

### トラブルシューティング経緯

**問題1：`next: not found`（exit code 127）**
- 原因：root の `netlify.toml` が `base="app"` を指定しているのに `app/netlify.toml` も読まれていた
- 対処：`app/netlify.toml` に `[build]` セクションを復活させ、両サイトで独立して機能する構成に整理

**問題2：`0 new function(s) to upload` → 本番 URL が 404**
- 原因：`@netlify/plugin-nextjs` が未インストール。静的ファイルのみ配信されていた
- 対処：`app/package.json` に `@netlify/plugin-nextjs` を追加し、`app/netlify.toml` に `[[plugins]]` ブロックを追記

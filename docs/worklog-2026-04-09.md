## Git 初期化の手順整理（未実施・次回以降）

**前提**：`ai-tool-diagnosis/` がプロジェクトルートとなる。
`app/` ディレクトリが Next.js プロジェクト本体。

```bash
# 1. プロジェクトルートに移動
cd ai-tool-diagnosis/

# 2. Git 初期化
git init

# 3. .gitignore の確認・調整
#    - app/node_modules/ ← 既存 .gitignore に /node_modules がある（app/ 配下なので要確認）
#    - app/.next/ ← 既存 .gitignore に /.next/ がある（同上）
#    - docs/ 以下は全て管理対象
#    → プロジェクトルート用 .gitignore を別途作成すること

# 4. 初回コミット
git add .
git commit -m "feat: initial commit — Next.js診断サイトMVP + PDF保存機能"

# 5. GitHub リポジトリ作成 → push（Phase 7 で実施）
```

**注意点**：
- `app/` に `.gitignore` が存在するが、これは Next.js デフォルトで `app/` 配下の `node_modules/` 等を対象にしたもの
- プロジェクトルート（`ai-tool-diagnosis/`）に別途 `.gitignore` を置くか、既存のものを活用するか要判断
- `docs/` はそのまま管理対象（Obsidian Vault 内なので `.obsidian/` 等の除外が必要な場合は要確認）
